# cp — Witchgrid control plane

The control-plane daemon. Owns the node registry, **profiles**, capacity-aware
placement, the routing API, the model catalog/transfer orchestration, the
dashboard, and GGUF introspection. Talks to one or more `agent/` instances over
plain HTTP. Compiles to a single ~7 MB binary (`witchgrid-cp`) with universal
dynamic deps.

## What's in here

| File | Role |
|---|---|
| `cp.hml` | Entry point. Opens sqlite (WAL + `busy_timeout`), seeds default profiles, starts the HTTP server + backup loop, dispatches routes. |
| `http_server.hml` | Minimal HTTP/1.1 server on `@stdlib/net.TcpListener`. Exact `(method, path)` routes + `*`-prefix slots; supports streaming a response from an upstream (`stream_upstream`) for SSE passthrough. No keep-alive, no TLS. |
| `gguf.hml` | GGUF v3 metadata parser → model size + KV-cache footprint for placement (`read_metadata`, `estimate_vram_mb`, `safe_context`, …). Unit-tested in `tests/gguf_test.hml`. |
| `inspect.hml` | Standalone `witchgrid-inspect <model.gguf>` CLI over the parser — "why won't this fit?" debugging. |
| `dashboard.hml` | HTMX fragment renderers (`/ui/nodes`, `/ui/services`, `/ui/profiles`, `/ui/capabilities`, `/ui/catalog`, `/ui/transfers`, `/ui/bench`, `/ui/stats`) + the shell at `/`. |
| `embedded_assets.hml` | Generated by `../scripts/embed_assets.sh` — dashboard + logo baked in as constants so the deploy artifact stays a single file. |

## Endpoints

```
# ── node registry ──────────────────────────────────────────────
POST /register        { node_id, hostname, role, hardware, agent_url, models?,
                        runtime?, unmanaged_processes?, auto_restart? }      → 200
GET  /nodes                                                                 → [...]  (public)
PATCH /api/nodes/{id}            { state: active|disabled|draining }        → 200
PUT  /api/nodes/{id}/auto-restart { auto_restart: bool }                    → proxies to agent

# ── services (proxied to the owning agent) ─────────────────────
POST /services        { profile, node_id?, model?, port?, gpus? }           → 201
GET  /services                                                              → [...]
POST /services/stop   { id, node_id }                                       → 204
GET  /services/log/{id}                                                     → text/plain (public)

# ── routing ────────────────────────────────────────────────────
GET  /resolve/{profile}                                                     → { host, port, base_url }  (public)
*    /v1/llama/{profile}/{path}                                             → proxied; auto-spawns if none running;
#       e.g. POST /v1/llama/chat-mahou/completion        stream:true → SSE passthrough

# ── profiles + aliases (CP-owned) ──────────────────────────────
GET/POST /api/profiles · GET/PUT/DELETE /api/profiles/{name}
GET/PUT /api/profile-aliases · DELETE /api/profile-aliases/{name}
GET  /profiles                                                              → [names]
GET  /capabilities | /api/capabilities                                     → { node: caps }

# ── model catalog / transfer ───────────────────────────────────
GET  /api/catalog · POST /api/placement_preview
POST /api/hf/search · POST /api/hf/files · POST /models/pull_hf · POST /models/distribute
GET  /api/transfers · GET /ui/missing

# ── settings · bench · observability · dashboard ──────────────
GET/PUT /api/settings            (db_backup_interval_min, db_backup_dir, db_backup_keep, …)
GET/POST /api/bench · GET /api/bench/{id}
GET  /metrics                                                              → Prometheus text  (public)
GET  /healthz                                                             → ok  (public)
GET  /  ·  /assets/*  ·  /ui/*                                            → dashboard (public)
```

`POST /services` **without** `node_id` triggers capacity-aware placement: CP
picks the (node, GPU set) with the model on disk + enough free VRAM, then proxies
the spawn. **With** `node_id` it forwards verbatim — required for models the GGUF
placement parser can't size (SD/whisper/piper), which spawn pinned with
`node_id` + `gpus` + `port` + `model`.

`/resolve` is a pure locator (never spawns) for resolve-then-connect consumers
that talk to the engine directly; `/v1/llama` is the full proxy that auto-spawns
and health-gates before completing.

## Build & run

```bash
../scripts/embed_assets.sh        # only if you changed a dashboard asset

hemlock cp.hml                    # interpreted, for dev iteration
hemlockc cp.hml -o witchgrid-cp && WITCHGRID_DATA_DIR=$HOME/.witchgrid ./witchgrid-cp
```

Binds `0.0.0.0:8765` (not yet env-configurable). State persists in
`witchgrid.db` (sqlite, WAL). **Set `WITCHGRID_DATA_DIR`** to pin the DB to an
absolute path; otherwise it's CWD-relative and the CP warns
`creating NEW database at …` when it makes a fresh one. Full deploy/ops:
[`../docs/operations.md`](../docs/operations.md).

## Smoke test (one agent registered)

```bash
curl -s http://localhost:8765/nodes | jq                       # who's in the grid
curl -s http://localhost:8765/metrics                          # Prometheus scrape

# auto-place + auto-spawn + complete in one shot
curl -X POST http://localhost:8765/v1/llama/chat-mahou/completion \
  -H 'content-type: application/json' \
  -d '{"prompt":"status?","n_predict":40}'

# SSE
curl -N -X POST http://localhost:8765/v1/llama/chat-mahou/completion \
  -H 'content-type: application/json' -d '{"prompt":"count","stream":true}'

# what's running, then stop it
curl -s http://localhost:8765/services | jq
ID=$(curl -s http://localhost:8765/services | jq -r '.[0].id')
NODE=$(curl -s http://localhost:8765/services | jq -r '.[0].node_id')
curl -X POST http://localhost:8765/services/stop -H 'content-type: application/json' \
  -d "{\"id\":\"$ID\",\"node_id\":\"$NODE\"}"
```
